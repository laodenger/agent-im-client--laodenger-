

\## 一、项目简介

本项目是\*\*前后端分离的本地 AI 聊天助手\*\*，实现了类商用聊天软件的完整交互体验。

用户可创建多个独立聊天窗口，AI 具备上下文记忆能力，支持\*\*自动判断并开启联网搜索\*\*，可处理实时信息查询、知识问答、计算、代码执行等多种场景，所有聊天记录本地永久保存，保护隐私。



\---



\## 二、技术选型理由

\### 后端技术

\- \*\*Python 3.11\*\*：AI 生态完善、开发高效、兼容性稳定

\- \*\*Flask\*\*：轻量 Web 框架，快速搭建 API 与 SSE 流式接口

\- \*\*LangChain 1.2.15\*\*：大模型交互标准框架，支持上下文管理与流式输出

\- \*\*DeepSeek Reasoner\*\*：推理能力强，支持工具调用与官方联网搜索

\- \*\*SQLite3\*\*：无需部署、文件型本地数据库，实现聊天记录持久化



\### 前端技术

\- \*\*原生 HTML/CSS/JS\*\*：无框架依赖、轻量、打开即用

\- \*\*SSE 服务器推送\*\*：实现 AI 打字机流式输出，体验流畅

\- \*\*多会话状态管理\*\*：独立聊天窗口、切换不中断、输出不丢失



\### 选型优势

\- 开箱即用，无需配置环境与数据库

\- 本地运行，聊天记录不上云

\- 多会话独立记忆，互不干扰

\- 支持工具自动调用，扩展能力强

\- 交互接近主流聊天产品



\---



\## 三、架构说明

```

用户浏览器（前端）

&#x20;     ↓ POST（消息 + 会话ID + 工具选择）

Flask 后端服务

&#x20;     ↓

LangChain 上下文管理 → 多会话独立记忆

&#x20;     ↓

DeepSeek 模型接口 → 流式输出 / 联网搜索

&#x20;     ↓

SQLite 本地数据库 → 聊天记录永久保存

```



\### 核心模块

1\. \*\*会话管理模块\*\*：多窗口独立 ID，支持新建/切换/删除

2\. \*\*上下文记忆模块\*\*：多轮对话保持连贯

3\. \*\*本地持久化模块\*\*：SQLite 保存所有会话记录

4\. \*\*流式输出模块\*\*：SSE 实时推送，逐字显示

5\. \*\*工具控制模块\*\*：根据用户问题自动判断是否开启联网搜索

\## 四、启动步骤

\### 1. 安装依赖

```bash

pip install flask flask-cors langchain langchain-openai

```



\### 2. 配置 API Key

打开 `main.py`，填入你的 DeepSeek API Key：

```python

DEEPSEEK\_API\_KEY = "你的 API Key"

```



\### 3. 启动后端服务

```bash

python main.py

```



\### 4. 启动前端

直接在浏览器打开 `index.html` 即可使用。



\### 5. 测试服务是否正常

访问：

```

http://127.0.0.1:5000/test

```

返回 `{"status":"ok"}` 即启动成功。
